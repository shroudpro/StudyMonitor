# StudyMonitor 测试版 v1 需求分析和概要设计

## 一、系统定位
StudyMonitor 是一个本地优先、开源、无侵入的智能学习行为监控系统。它通过计算机视觉技术（YOLO Pose）和本地大语言模型（Ollama + Qwen2.5），在用户的个人设备上提供实时的学习状态监控、行为解释，和基于自然语言驱动的规则引擎，助力用户养成良好专注的学习习惯。
在测试版 v1 中，系统完成了从 MVP（最小可用产品）向正式测试产品的转变。

核心创新：
1. **纯本地离线执行**：全流程包括计算机视觉和自然语言处理在本地设备进行，极致保护用户隐私。
2. **规则导向 + 智能辅助**：系统依然基于确定的条件规则引擎进行四状态判定，但引入本地小参数 VLM 实现规则与行为的自然语言转换。
3. **极简优雅设计**：抛弃繁复的大屏数据面板，采用苹果风格/极简实用美学的 Design System。

## 三、核心功能流改造

在 v1 测试版中，功能流的核心亮点不再是单纯的视觉跟踪，而是引入了两种基于 VLM（Qwen2.5-1.5B）的核心增强：

### 3.1 VLM 辅助状态解释 (Semantic)
通过状态变化事件被动触发（不在每帧触发，大幅节约性能消耗）。
**输入：**
- 判定出的四类结果（专注、分心、低效、离开）
- 底层抽象状态的组合参数（是否在场？是否偏头？是否低头？姿态稳定度？）
- 当前匹配规则标签

**输出**：稳定友好的一句话概括总结。并记录在**历史记录栈**中供前端显示时间线。

### 3.2 VLM 辅助规则生成
系统具备内置的默认规则。但当用户习惯不同的触发阈值时（比如喜欢用电脑时来回扭动不算分心），支持自然语言改写。
前端 UI 新增 "AI 规则生成" 功能。
输入："低头超过6秒判定为低效"。
VLM 模型将自然语言转译为 JSON 格式规则结构：
```json
{
  "ruleName": "rule_head_down_6s",
  "conditionJson": "{\"head_down\": true, \"duration_sec\": {\">\": 6}}",
  "outputState": "低效"
}
```

### 3.3 默认状态判定引擎优化
为了改善实际使用中由于微小动作带来的误判，v1 版的引擎判断逻辑做了防抖加宽处理：

| 优先级 | 判定状态 | 核心逻辑条件 (Python) | 优化点说明 |
| :--- | :--- | :--- | :--- |
| **1** | **离开 (AWAY)** | `not is_present and away_duration > 5.0` | **防抖延长**：将时间从 3s 提升到 5s，防止用户弯腰捡笔或伸懒腰短暂离开被误判。 |
| **2** | **分心 (DISTRACTED)** | `is_present and head_turned_away and not head_down and stable_duration > 10.0` | **剔除低头**：增加了 `not head_down` 的限制。只有在“非低头”状态下长期偏头（东张西望）才算分心。时间放宽至 10s。 |
| **3** | **专注 (FOCUS)** | **A. 伏案专注 (优先)：** `is_present and head_down and inactive_duration <= 60.0`<br>**B. 屏幕专注：** `is_present and not head_down and not head_turned_away and inactive_duration <= 30.0` | **双轨判定 (核心)**：1. 低头时（写字/看书）允许长达 60s 静止。2. 抬头看屏幕时允许 30s 静止。 |
| **4** | **低效 (LOW)** | 兜底状态 (当上述 1、2、3 均不满足时落入此状态)。<br>触发场景举例：<br>- 抬头盯屏幕发呆超过 30s<br>- 低头睡着或完全僵直超过 60s<br>- 其他无法归类的异常姿态 | **缩小打击面**：只惩罚“真正长时间毫无微动”的极端静止状态。只要有微小动作打断计时，就不会落入兜底。 |

## 四、数据模型变更

测试版 v1 针对数据追踪补充了新字段：
1. `BehaviorLog`：未变
2. `BehaviorRule`：未变
3. `StudySession`：未变
4. `SemanticLog`：
   - 增加 `sessionId` 以关联学习周期
   - 增加 `source` 以区分是由 `vlm` 生成还是因为模型宕机由 `template` 生成
5. `NlRuleParseLog` (新增)：
   - 记录每次自然语言指令、转译 JSON 结果、成功或失败情况，供后续模型微调使用。

## 五、环境部署方案

针对最终用户的一键部署方案优化（避免由于安装多套框架导致的路径不一致）：

1. **环境包依赖**
   使用 `Conda` 进行隔离管理：
   ```bash
   conda create -n study-monitor python=3.11
   conda activate study-monitor
   pip install fastapi uvicorn[standard] opencv-python-headless onnxruntime sqlalchemy pydantic websockets python-multipart httpx
   ```

2. **本地模型挂载**
   - 机器视觉：直接部署 YOLO `onnx` 到 `/models` 文件夹下。
   - 大规模认知：通过官方方式安装 Ollama 守护线程。通过 `ollama run qwen2.5:1.5b` 同步轻量级量化模型。

3. **前后端启动**
   - 客户端：`npm install` + `npm run dev`
   - 服务端：`uvicorn app.main:app --host 0.0.0.0 --port 8000`

## 六、未来规划 (Post-v1)

1. **数据分析与周报**：当前只有单次 session 级别的数据聚类，后续将接入持久化的长期周报分析。
2. **主动式干预**：加入语音播报 / Mac 或 Windows 通知中心集成。
3. **更轻的模型接入层**：考虑将 VLM 使用 ONNX 直接接入，不需要依赖 Ollama 提供端点。
